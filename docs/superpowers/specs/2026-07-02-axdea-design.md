# AXdea — 아이디어 놀이터 설계 문서

- 작성일: 2026-07-02
- 작성자: AX기획실 (요청: 박찬영 부장)
- 목적: 하반기 "몰입의 날" 파일럿을 위해, 팀원이 각자의 문제의식·아이디어를
  가볍게 올리고 공유하는 플레이풀한 아이디어 보드

## 1. 한 줄 요약

아이디어 하나가 DiceBear 캐릭터 1마리가 되어 화면을 탱탱볼처럼 튕겨다니고,
캐릭터를 클릭하면 아이디어 카드가 열리는 단일 화면 물리 캔버스 웹앱.
작성자는 기록되지만 기본 숨김이며, 이름을 "박찬영"으로 입력한 세션에서만
모든 작성자 이름이 드러난다.

## 2. 컨셉 & 화면 구성

- **단일 화면 물리 캔버스**가 메인 무대.
- 아이디어 1개 = **DiceBear 캐릭터 1마리**.
  - 캐릭터는 위치/속도를 가지고 화면을 떠다닌다: 벽 충돌 반사, 서로 부드럽게 밀침.
  - 드래그로 잡아 던지면 관성으로 튕기고, 놓으면 다시 굴러다닌다.
  - 클릭(짧게 탭)하면 아이디어 카드가 열린다. 드래그와 클릭은 이동 거리로 구분.
- **아이디어 카드(패널/모달)**: 제목, 내용, 카테고리, 색상, 댓글 목록/입력.
- **`+` 플로팅 버튼**(우하단): 새 아이디어 작성 → 저장 시 새 캐릭터가 화면에 합류.
- **시각 신호**:
  - 캐릭터의 색상 아우라 = 아이디어 색상.
  - 작은 배지 = 카테고리.
  - 멀리서도 카테고리/색 분포가 한눈에 보인다.
- **카테고리 필터 바**(상단): 특정 카테고리만 강조/디밍.
- 라이트/다크 모드 대응.

### DiceBear
- HTTP API 사용: `https://api.dicebear.com/9.x/{style}/svg?seed={seed}`
- 재미있는 스타일 세트에서 랜덤 배정: `fun-emoji`, `bottts`, `adventurer`,
  `thumbs`, `open-peeps`, `big-smile` 등.
- 스타일+seed를 아이디어에 저장 → 항상 같은 캐릭터로 재현.
- 매번 랜덤이라 작성자 추측이 어려워 익명성에도 유리.

## 3. 기술 스택 & 호스팅 (rhythm 프로젝트와 동일 방식)

- **순수 HTML / CSS / JS** (빌드 단계 없음).
- **Supabase**: 브라우저에서 anon key로 직접 접근 (supabase-js CDN).
- **GitHub Pages** 정적 호스팅.
- 파일 구조:
  - `index.html` — 마크업
  - `styles.css` — Twenty 기반 스타일 + 캔버스/캐릭터 스타일, 라이트·다크
  - `app.js` — 상태 / Supabase / 물리 엔진 / 렌더 / 이벤트 / 모달
  - `supabase.sql` — 테이블 + RLS 생성 스크립트
  - `README.md` — 셋업·배포 가이드
  - `.claude/settings.json` — 턴 종료 시 자동 커밋·푸시 훅(rhythm과 동일)

## 4. 데이터 모델 (Supabase)

### 테이블: `ideas`
| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | uuid (default gen_random_uuid()) | PK |
| title | text | 제목 |
| body | text | 내용 |
| category | text | 카테고리 키 |
| color | text | 색상 키/HEX |
| avatar_style | text | DiceBear 스타일 |
| avatar_seed | text | DiceBear seed |
| author | text | 작성자 (기록하되 기본 숨김) |
| created_at | timestamptz (default now()) | |

### 테이블: `comments`
| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | uuid (default gen_random_uuid()) | PK |
| idea_id | uuid (FK → ideas.id, on delete cascade) | |
| author | text | 작성자 (기본 숨김) |
| body | text | 댓글 내용 |
| created_at | timestamptz (default now()) | |

### RLS 정책 (내부 신뢰 기반)
- RLS 활성화.
- anon 역할에 대해 `select`, `insert` 허용.
- `update`/`delete`: 본인 노트 삭제 정도만 필요하면 이후 추가. 파일럿에선
  삭제는 열람 모드(박찬영)에서만 UI 노출하는 것으로 단순화. (DB 레벨 강제는 안 함.)

## 5. 익명 & 권한 흐름 (확정: 이름만으로)

1. 첫 진입 → **이름 입력 모달**. 입력값을 `localStorage`에 저장, 이후 작성하는
   `ideas.author` / `comments.author`에 기록.
2. 기본 상태: 모든 아이디어/댓글의 **작성자 미표시**(캐릭터만, 라벨 없음).
3. 입력한 이름이 정확히 **"박찬영"**이면 → **전체 열람 모드**:
   모든 캐릭터/카드/댓글에 작성자 이름 라벨 표시.
4. 열람 모드에서만 아이디어 삭제 버튼 노출(관리 편의).
5. 세션 이름은 헤더에서 언제든 변경 가능.

> 보안 주의(합의됨): 누구든 이름을 "박찬영"으로 입력하면 익명이 풀린다.
> 내부에서 단기간만 쓰는 파일럿이므로 PIN 없이 이대로 진행.

## 6. 디자인 (Twenty 기반)

- Twenty(twentyhq) 무드: 깔끔한 미니멀, 부드러운 회색/화이트 표면, 둥근 모서리,
  절제된 그림자, 선명한 타이포, 차분한 액센트 컬러.
- UI 크롬(헤더, 모달, 카드, 버튼, 필터 바)에 이 무드를 적용.
- 중앙 캔버스만 컬러풀·플레이풀하게 대비 → 도구는 진중, 놀이터는 발랄.
- 라이트/다크 대응.

## 7. 물리 엔진 (경량, 자체 구현)

- requestAnimationFrame 루프.
- 각 캐릭터: {x, y, vx, vy, r}. 매 프레임 위치 갱신, 벽 반사(속도 반전 + 감쇠),
  캐릭터 간 원-원 충돌 시 부드러운 반발.
- 마찰(감쇠)로 서서히 느려지되 완전히 멈추지 않게 미세 지터/최소 속도 유지.
- 드래그: pointerdown으로 잡고 이동, pointerup 시 최근 이동 속도를 vx/vy로 부여(던지기).
- 드래그 이동 거리가 임계값 미만이면 클릭(카드 열기)으로 처리.
- 성능: 캐릭터 수가 많아도 부드럽게(수십 마리 기준). DiceBear는 <img>로 1회 로드 후 DOM transform으로 이동.

## 9. 진행에 필요한 외부 입력

- Supabase **새 프로젝트(AXdea)** 생성 후 `Project URL` + `anon key` 제공(부장님 1회).
- GitHub 저장소 오너/이름 확정(예: `rejoyful/AXdea`).

## 10. 성공 기준

- 이름 입력 후 즉시 캔버스 진입, 캐릭터들이 자연스럽게 튕겨다닌다.
- 누구나 30초 안에 아이디어 1개를 올릴 수 있다.
- 아이디어는 익명으로 공유되고, "박찬영" 세션에서만 작성자가 드러난다.
- 카테고리/색으로 아이디어 분포가 한눈에 읽힌다.
- GitHub Pages URL 하나로 팀 전체가 접속 가능.
