# AXdea · 아이디어 놀이터

팀원들의 아이디어를 **DiceBear 캐릭터**로 띄우는 물리 캔버스 웹앱. 캐릭터가 화면을 탱탱볼처럼 튕겨다니고,
클릭하면 아이디어 카드가 열립니다. 작성자는 기본 **가려지며**, **관리자(박찬영·이해원)** 로 입장한 세션에서만 전체 작성자가 보입니다.
네온 전광판 UI · 좋아요/커피/해보자/아쉬워 · 대댓글 · 팀장 Pick 메달 · 선정→복제 파이프라인 · 라운드 아카이브 · 분할 비교 포함.

---

## 사용법 (기능별)

### 0. 입장 — 이름 + 입장 코드
접속하면 **이름**과 **입장 코드**를 입력해야 들어갑니다(본인확인).
- **일반 멤버**: 이름 + **팀 공용 코드** → 입장(작성자 익명)
- **관리자(박찬영·이해원)**: 각자 이름 + **본인 관리자 코드** → 입장 + **전체열람·관리 기능**
- 코드 값은 팀 내부에서만 공유합니다. 코드는 `apps/web/public/js/config.js` 에 **SHA-256 해시로만** 저장되며(원문 미저장), 변경은 새 코드의 해시로 교체.

> 권한 요약: **일반 멤버**(작성·좋아요·댓글·**새 라운드 시작**) → **관리자**(전체 작성자 열람·반려·선정복제·라운드 이름변경/삭제) → **팀장(박찬영)**(팀장 Pick 메달 부여).

### 1. 아이디어 올리기
- 우측 하단 **＋** 버튼으로 작성: 한 줄 제목 · 내용 · 카테고리 · 색상 (아바타는 자동 배정).
- 올린 아이디어는 **캐릭터(오브제)** 로 등장해 캔버스를 튕겨다닙니다. **드래그로 옮기거나 던질** 수 있어요.
- 캐릭터를 **클릭**하면 상세 카드가 열립니다.
- **본인 글**은 위에 **내 이름이 잔잔히 깜빡**여 구분됩니다(관리자는 전체 이름 표시, 그 외엔 남의 글은 익명).

### 2. 반응 — 좋아요 / 커피 / 해보자 / 아쉬워 / 댓글
- **좋아요**: 카드에서 하트, **1인 1회**(다시 누르면 취소). 좋아요가 쌓이면 캐릭터가 커지고, **네온 광선(인챈트) 궤도**가 늘어납니다(2개=1줄, 5개=2줄, 9개=3줄).
- **댓글 / 답글(대댓글)**: 카드 하단에서 작성. 댓글에 **태그**를 달 수 있어요 — **해보자 · 아쉬워 · 커피**(“내가 커피 살게” 같은 맥락을 남기는 용도, 3택 중 하나).

### 3. 팀장 Pick (메달)
- **팀장(박찬영)** 만 카드의 **`팀장 Pick`** 버튼을 누를 수 있습니다.
- Pick하면 그 아이디어 **오브제 좌상단에 반짝이는 골드 메달**이 붙어 모두에게 보입니다(다시 누르면 해제).

### 4. 선정 → 라운드로 복제 (실행 파이프라인)
- 관리자는 카드에서 **`선정 · 라운드로 복제`** → **대상 라운드 선택**(기존 라운드 또는 새로 만들기).
- 원본에는 **‘선정’ 표시**가 붙고, **복제본은 반응이 초기화**되어 실행 라운드에서 새로 피드백(해보자/아쉬워/커피)을 받습니다.

### 5. 라운드(주제/세션) & 아카이브
- 상단 **LED 전광판**이 현재 보고 있는 라운드를 표시(진행 중=LIVE, 지난 라운드=ARCHIVE 읽기전용).
- **＋ 새 라운드**(헤더 우측, **모든 사용자**): 현재 라운드를 아카이브로 보관하고 새 라운드 시작.
- **🗂 아카이브**: 라운드를 **카드**로 모아 봄 — 카드 클릭 시 열람, **이름 변경**(관리자), **삭제**(관리자 · 휴지통, **진행 중 라운드는 삭제 불가**. 삭제 시 그 라운드 아이디어·댓글·반응 영구 삭제).

### 6. 전체 목록 & 분할 비교
- **≣ 목록**: 전체 아이디어를 **카드 그리드(한 줄 5개)** 로 보기 + **정렬**(좋아요/커피/해보자/아쉬워/댓글/이름순).
- **▤ 분할**: **2~4개 라운드를 카드로 선택**해 화면을 나눠 나란히 비교(읽기전용 · 데스크톱·태블릿 전용).

### 7. 그 외
- **테마**: 헤더에서 **다크(기본)/라이트** 토글.
- **고양이 5마리**(느긋이·폴짝이·새침이·응석이·부끄럼): 각자 성격대로 아이디어 위를 돌아다니는 장식.
- **실시간**: 4초 폴링으로 다른 사람의 반응/댓글이 자동 반영.

## 구조 (모노레포 · Next.js)

```
AXdea/
├─ apps/web/                 # Next.js 앱 (프론트 + /api) = 배포 단위
│  ├─ app/
│  │  ├─ page.js             # 기존 캔버스 UI(바닐라 JS)를 그대로 감싼 페이지
│  │  ├─ layout.js, globals.css
│  │  ├─ lib/db.js           # MySQL 접속 풀
│  │  └─ api/**/route.js     # REST API (MySQL) — Supabase/Express 대체
│  ├─ public/js/{app,pure,config,api}.js   # 앱 로직(바닐라 JS)
│  └─ db/schema.sql          # (신규 세팅용) MySQL 스키마
├─ ecosystem.config.js       # PM2 설정 (deploy.sh용, 앱 이름 web)
├─ .env.example              # 루트 .env 템플릿 (DB 접속정보)
└─ deploy.sh                 # 표준 배포 스크립트(사내) — 루트에 배치해 실행
```

- **웹서버(Next)**: `192.168.100.105:5114` — 앱 + `/api`를 한 프로세스가 서빙 (같은 오리진 → CORS 불필요)
- **DB서버(MySQL)**: `192.168.100.76:3306`, DB명 `axdea` (**웹서버와 다른 호스트**)
- **도메인**: `axdea.hakjisa.kr` (사내 상위 nginx가 5114로 프록시)
- **실시간**: 4초 폴링

## 배포 (사내 표준 deploy.sh 사용)

192.168.100.105 웹서버에서:
```
git clone https://github.com/rejoyful/AXdea.git
cd AXdea
cp .env.example .env         # DB_PASSWORD 등 채우기 (PORT=5114, DB_HOST=192.168.100.76, DB_PORT=3306)
bash deploy.sh               # git pull → npm install → next build → PM2(web:5114) 재기동
```
`deploy.sh` 가 `apps/web` 을 빌드하고 PM2 `web`(포트 5114)로 상시 구동합니다. 도메인/HTTPS는 사내 상위 nginx가 5114로 프록시.

## 로컬 개발
```
cd apps/web
npm install
DB_PASSWORD=... npm run dev      # http://localhost:5114 (DB는 원격 MySQL)
```
API가 안 붙으면(오프라인 등) 앱은 자동으로 로컬 데모 모드로 동작합니다.

## 참고
- 데이터는 Supabase → MySQL 이전 완료(이 저장소는 MySQL 백엔드 기준).
- **스키마 자동 보강**: 앱 부팅 시 `lib/db.js`의 `ensureSchema()`가 필요한 컬럼/인덱스를 자동 추가합니다. 배포만 하면 DB 수동 작업(ALTER) 없이 최신 스키마가 적용됩니다.
- **설정 위치**: 관리자 이름/팀장/입장 코드 해시 → `apps/web/public/js/config.js`. 아바타 스타일·카테고리·색상 팔레트도 같은 파일.
- 로직 단위 테스트: `cd apps/web && npm test`.
